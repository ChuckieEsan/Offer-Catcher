## 优化内容

### 1. 短期记忆优化 - 消息裁剪

**问题**: 固定 10 条消息窗口可能导致 Token 超限或丢失重要上下文

**解决方案**: `_apply_message_trimming()` 函数

**文件**: `backend/app/agents/graph/nodes.py`

```python
def _apply_message_trimming(
    messages: list[BaseMessage],
    max_messages: int = 10,
    max_tokens: int = 8000
) -> list[BaseMessage]:
    """应用消息裁剪策略

    策略：
    1. 保留 SystemMessage（如果有）
    2. 保留最近 N 条消息
    3. 如果 token 超限，进一步裁剪
    """
```

**关键逻辑**:
- 分离 system message 和普通消息
- 优先保留最近的对话
- Token 超限时自动裁剪到一半

### 2. 长期记忆模块

**文件**: `backend/app/memory/` (新建目录)

```
memory/
├── __init__.py
└── long_term.py
```

#### 数据模型

| 模型 | 用途 | 字段 |
|------|------|------|
| `UserProfile` | 用户画像 | preferred_companies, target_position, tech_stack, experience_years |
| `UserPreferences` | 偏好设置 | language, difficulty, practice_batch_size, show_answer_immediately |
| `LearningProgress` | 学习进度 | mastered_entities, pending_review_question_ids, total_questions_answered |
| `SessionSummary` | 会话摘要 | session_id, summary, key_topics, created_at |

#### 存储结构

```
PostgreSQL (langgraph_store)
├── ("users", user_id, "profile") → UserProfile
├── ("users", user_id, "preferences") → UserPreferences
├── ("users", user_id, "progress") → LearningProgress
└── ("users", user_id, "summaries", session_id) → SessionSummary
```

### 3. 主流程集成

#### main.py - 应用启动时初始化

```python
@asynccontextmanager
async def lifespan(app: FastAPI):
    # ... 其他初始化 ...
    
    # 初始化长期记忆存储
    from app.memory import get_long_term_memory
    try:
        memory = get_long_term_memory()
        if memory.initialized:
            logger.info("Long-term memory initialized")
        else:
            logger.warning("Long-term memory not fully initialized")
    except Exception as e:
        logger.warning(f"Long-term memory init failed: {e}")
```

#### workflow.py - 支持 user_id 参数

```python
async def astream_workflow(
    messages: list[BaseMessage],
    thread_id: Optional[str] = None,
    user_id: Optional[str] = None,  # 新增
    **kwargs
) -> AsyncGenerator[dict, None]:
    # 将 user_id 放入 context 中
    if user_id:
        initial_state["context"] = {"user_id": user_id}
```

#### nodes.py - 检索并注入用户上下文

```python
@traced_async
async def react_loop_node(state: AgentState, config: RunnableConfig) -> AgentState:
    # ... 消息裁剪 ...
    
    # 获取用户 ID 并检索长期记忆上下文
    context = state.get("context", {})
    user_id = context.get("user_id")
    if user_id:
        try:
            from app.memory import get_user_context_prompt
            user_context = get_user_context_prompt(user_id)
            # 将用户上下文插入到 system message 中
            for i, msg in enumerate(messages):
                if getattr(msg, "type", "") == "system":
                    original_content = getattr(msg, "content", "")
                    new_content = original_content + "\n\n" + user_context
                    messages[i] = msg.__class__(content=new_content)
                    break
        except Exception as e:
            logger.warning(f"Failed to inject user context: {e}")
```

#### chat_agent.py - 传递 user_id

```python
async def achat_streaming(
    self,
    message: str,
    conversation_id: str,
    user_id: Optional[str] = None,  # 新增
) -> AsyncGenerator[str, None]:
    async for event in astream_workflow(
        messages=[HumanMessage(content=message)],
        thread_id=conversation_id,
        user_id=user_id,  # 传入 user_id
    ):
```

#### chat.py - API 接口

```python
class ChatRequest(BaseModel):

@router.post("/stream")
```
