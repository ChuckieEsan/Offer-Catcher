## 2. Agent 系统设计

### 2.1 BaseAgent 基类设计

`BaseAgent` 提供了所有 Agent 的通用能力，采用泛型设计支持不同的输出类型：

```python
class BaseAgent(Generic[T]):
    """Agent 基类
    
    提供以下通用能力：
    - LLM 初始化（支持多 Provider）
    - Structured Output 支持
    - Prompt 模板加载（支持 jinja2 格式）
    - 重试机制（默认启用）
    """
    
    _prompt_filename: str = ""
    _structured_output_schema: Optional[type] = None

    def __init__(self, provider: str = "deepseek", llm_kwargs: Optional[dict] = None):
        self._llm = create_llm(self.provider, "chat", **self._llm_kwargs)
        self._structured_llm = None

    @property
    def structured_llm(self) -> Optional[ChatOpenAI]:
        """延迟加载 Structured Output LLM"""
        if self._structured_llm is None:
            self._structured_llm = self.llm.with_structured_output(
                self._structured_output_schema,
                method="function_calling"
            )
        return self._structured_llm

    @retry(max_retries=3, delay=1.0, backoff=2.0)
    def invoke_llm(self, prompt: str) -> str:
        """带重试的 LLM 调用"""
        return self.llm.invoke(prompt).content
```

**设计亮点**：
- **泛型支持**：`Generic[T]` 允许子类指定结构化输出类型
- **延迟加载**：`structured_llm` 按需初始化，避免不必要的资源消耗
- **重试机制**：通过装饰器实现指数退避重试，提高调用成功率

### 2.2 LangGraph 工作流设计

项目使用 LangGraph 构建 Agent 工作流，实现复杂的多步骤任务编排：

```python
# 工作流结构
START → state_gate → (router / handle_confirmation)
router → (extract / react_loop)
extract → confirm → END
handle_confirmation → (store_and_mq / extract / END)
store_and_mq → END
react_loop → END
```

**核心节点说明**：

| 节点 | 职责 |
|------|------|
| `state_gate` | 状态门，检查是否有待处理的确认流程 |
| `router` | 意图分类，决定进入 ingest 还是 react 流程 |
| `extract` | 调用 Vision Extractor 提取面经 |
| `confirm` | 展示提取结果，等待用户确认 |
| `react_loop` | ReAct 循环，处理用户查询 |

```python
def _apply_message_trimming(
    messages: list[BaseMessage],
    max_messages: int = 10,
    max_tokens: int = 8000
) -> list[BaseMessage]:
    """应用消息裁剪策略
    
    策略：
    1. 保留 SystemMessage
    2. 保留最近 N 条消息
    3. 如果 token 超限，进一步裁剪
    """
    # 分离 system message 和普通消息
    system_messages = [m for m in messages if m.type == "system"]
    other_messages = [m for m in messages if m.type != "system"]
    
    # 保留最近的 N 条
    trimmed = other_messages[-max_messages:]
    
    if system_messages:
        trimmed = [system_messages[0]] + trimmed
    
    return trimmed
```

**优势**：
- 语义检索更准确，区分不同公司/岗位的同名题目

### 3.2 混合检索：向量 + Payload 过滤

Qdrant 支持 Payload 预过滤（Pre-filtering），先过滤再计算向量相似度：

```python
def _build_search_filter(self, filter_conditions: Optional[SearchFilter]) -> Optional[models.Filter]:
    """构建搜索过滤条件"""
    if not filter_conditions:
        return None
    
    must_conditions = []
    
    # 公司过滤（精确匹配）
    if filter_conditions.company:
        must_conditions.append(
            models.FieldCondition(
                key="company",
                match=models.MatchValue(value=filter_conditions.company),
            )
        )
    
    # 知识点过滤（数组多值匹配）
    if filter_conditions.core_entities:
        must_conditions.append(
            models.FieldCondition(
                key="core_entities",
                match=models.MatchAny(any=filter_conditions.core_entities),
            )
        )
    
    return models.Filter(must=must_conditions)
```

### 3.3 分类熔断机制

不同类型的题目采用不同的处理策略：

```python
# 分类熔断逻辑
```

- `project` / `behavioral` 类型依赖个人经历，LLM 容易产生幻觉

### 3.5 主键幂等性设计

所有 `question_id` 使用 MD5 哈希生成，确保幂等性：

```python
def generate_question_id(company: str, question_text: str) -> str:
    """生成题目 ID
    
    使用 MD5("公司名" + "题目文本") 生成唯一标识
    """
    content = f"{company}{question_text}"
    return hashlib.md5(content.encode("utf-8")).hexdigest()
```

**优势**：
- 相同题目相同公司始终生成相同 ID
- 支持 Upsert 操作，避免重复入库

## 4. 消息队列异步处理

### 4.1 异步生产者设计

使用 `aio-pika` 实现异步消息发布，支持自动重连：

```python
class AsyncRabbitMQProducer:
    async def connect(self) -> bool:
        """建立与 RabbitMQ 的强健连接"""
        # connect_robust 会在网络抖动时自动重连
        self._connection = await aio_pika.connect_robust(
            host=self.settings.rabbitmq_host,
            port=self.settings.rabbitmq_port,
            login=self.settings.rabbitmq_user,
            password=self.settings.rabbitmq_password,
        )
        self._channel = await self._connection.channel()
        
        # 声明队列（确保队列存在）
        await self._channel.declare_queue(
            self.settings.rabbitmq_queue,
            durable=True,  # 持久化
        )

    async def publish_task(self, task: MQTaskMessage, retry: int = 3) -> bool:
        """发布单条任务消息（带重试）"""
        msg = Message(
            body=task.model_dump_json().encode("utf-8"),
            delivery_mode=DeliveryMode.PERSISTENT,  # 持久化消息
            message_id=task.question_id,
        )
        await self._exchange.publish(msg, routing_key=self.settings.rabbitmq_queue)
```

### 4.2 消费者熔断机制

消费者集成了断路器模式，防止级联故障：

```python
class AsyncRabbitMQConsumer:
    def __init__(self, prefetch_count: int = 5):
        self.circuit_breaker = create_circuit_breaker(
            fail_max=5,
            timeout_duration=30.0,
            name="rabbitmq_consumer",
        )

    async def _on_message(self, message: AbstractIncomingMessage, callback):
        # 熔断器拦截逻辑
        if isinstance(self.circuit_breaker.state, CircuitOpenState):
            if self._circuit_open_time and (time.time() - self._circuit_open_time >= self._recovery_timeout):
                self.circuit_breaker.close()  # 恢复
            else:
                # 正在熔断中：拒绝该消息使其重新排队
                await message.reject(requeue=True)
                return
        
        try:
            task = MQTaskMessage.model_validate_json(message.body)
            success = await callback(task)
            
            if success:
                await message.ack()
                self.circuit_breaker.close()  # 成功后关闭熔断器
            else:
                # 业务明确返回失败：重新投递到队尾
                await message.ack()
                await self.republish_to_back(message, question_id, retry_count)
                
        except Exception as e:
            self.circuit_breaker.open()  # 打开熔断器
            await message.reject(requeue=True)
```

### 4.3 死信队列 (DLQ) 设计

消费失败超过阈值的消息会被发送到死信队列：

```python
async def connect(self):
    # 声明主队列
    self._queue = await self._channel.declare_queue(
        self.settings.rabbitmq_queue, 
        durable=True
    )
    # 声明死信队列

    )
```

## 7. 缓存与单例设计

### 7.1 @cached 装饰器

根据函数参数自动缓存结果：

```python
def cached(func: Callable[..., T]) -> Callable[..., T]:
    """通用缓存装饰器
    
    自动根据函数参数生成缓存键，支持位置参数和关键字参数。
    """
    _cache: dict[str, T] = {}

    @wraps(func)
    def wrapper(*args, **kwargs) -> T:
        key = _make_key(args, kwargs)
        if key not in _cache:
            _cache[key] = func(*args, **kwargs)
        return _cache[key]

    wrapper.clear_cache = lambda: _cache.clear()
    return wrapper
```

### 7.2 @singleton 装饰器（线程安全）

单例模式装饰器，支持线程安全：

```python
def singleton(func: Callable[..., T]) -> Callable[..., T]:
    """单例缓存装饰器（线程安全）"""
    
    class Container:
        __slots__ = ('instance',)
        def __init__(self):
            self.instance: T | None = None

    _container = Container()
    _lock = threading.Lock()

    @wraps(func)
    def wrapper(*args, **kwargs) -> T:
        # 快速路径：已存在实例时直接返回
        if _container.instance is not None:
            return _container.instance

        # 慢速路径：需要创建实例，加锁保护
        with _lock:
            # 双重检查
            if _container.instance is None:
                _container.instance = func(*args, **kwargs)
            return _container.instance

    def clear_cache() -> None:
        with _lock:
            _container.instance = None

    wrapper.clear_cache = clear_cache
    return wrapper
```

**设计亮点**：
- 双重检查锁（Double-Checked Locking）减少锁竞争
- 使用容器类避免 `nonlocal` 与 `lambda` 的冲突
- 提供 `clear_cache()` 方法用于测试重置

### 7.4 工具层缓存设计

Agent 工具层集成了 Redis 缓存，减少重复的数据库查询和外部 API 调用：

```python
# search_questions 工具缓存集成

    # 构建缓存 key

```

**缓存策略设计**：

| 工具 | 缓存 TTL | 设计理由 |
|------|---------|---------|
| `search_questions` | 5 分钟 | 题库变化不频繁，向量检索成本高 |
| `query_graph` | 10 分钟 | 图数据变化慢，Neo4j 查询成本高 |
| `search_web` | 30 分钟 | Web 内容相对稳定，API 调用有成本 |

**缓存防护机制**：

```python
class CacheService:
    """缓存服务
    
    提供 Redis 缓存的统一管理，包括：
    - TTL 随机化（防雪崩）
    - 分布式锁（防击穿）
    - 缓存空值（防穿透）
    """

    def get_with_lock(self, key: str, fetch_fn: Callable, ttl: int) -> Any:
        """带分布式锁的缓存读取（防击穿）"""
        # 1. 查缓存
        hit, cached = self.get(key)
        if hit:
            return cached

        # 2. 获取分布式锁
        lock_value = self._acquire_lock(key)

        # 3. 双重检查
        hit, cached = self.get(key)
        if hit:
            return cached

        # 4. 查数据库
        data = fetch_fn()

        # 5. 写缓存（包括空值）
        self.set(key, data, ttl)

        return data
```

**缓存失效策略**：

```python
def invalidate_question(self, question_id: str = None):
    """失效题目相关缓存

    写操作后调用，确保缓存一致性。

    self.delete_pattern(CacheKeys.questions_list_pattern())

    # 2. 删除统计数据缓存
    self.delete(CacheKeys.stats_overview(), ...)

    # 3. 删除工具缓存

```

**设计亮点**：
- 自动缓存空值，防止缓存穿透
- 分布式锁保护，防止缓存击穿
- TTL 随机化，防止缓存雪崩
- 写操作自动失效相关缓存，保证一致性

### 8.2 追踪装饰器

提供 `@traced` 和 `@traced_async` 装饰器：

```python
def traced(name: Optional[str] = None):
    """同步函数追踪装饰器
    
    自动记录调用次数、时长、成功/失败状态。
    """
    def decorator(func: Callable[..., T]) -> Callable[..., T]:
        span_name = name or func.__name__

        @wraps(func)
        def wrapper(*args, **kwargs) -> T:
            with tracer.start_as_current_span(span_name) as span:
                start_time = time.perf_counter()
                try:
                    result = func(*args, **kwargs)
                    duration_ms = (time.perf_counter() - start_time) * 1000
                    
                    # 记录成功指标
                    _tool_call_counter.add(1, {"name": span_name, "status": "success"})
                    _tool_call_duration.record(duration_ms, {"name": span_name})
                    
                    span.set_attribute("status", "success")
                    return result
                except Exception as e:
                    _tool_call_errors.add(1, {"name": span_name, "error_type": type(e).__name__})
                    span.record_exception(e)
                    raise
        return wrapper
    return decorator
```

## 9. 防御性编程实践

### 9.1 重试装饰器

带指数退避的重试机制：

```python
def retry(max_retries: int = 3, delay: float = 1.0, backoff: float = 2.0):
    """重试装饰器"""
    def decorator(func: Callable):
        @wraps(func)
        def wrapper(*args, **kwargs):
            current_delay = delay
            for attempt in range(max_retries):
                try:
                    return func(*args, **kwargs)
                except exceptions as e:
                    if attempt < max_retries - 1:
                        logger.warning(f"{func.__name__} attempt {attempt + 1} failed, retrying...")
                        time.sleep(current_delay)
                        current_delay *= backoff
                    else:
                        raise
        return wrapper
    return decorator
```

### 9.2 断路器模式

基于 `aiobreaker` 实现断路器，防止级联故障：

```python
def circuit_breaker(fail_max: int = 5, timeout_duration: float = 30.0):
    """断路器装饰器
    
    工作原理：
    - 连续失败达到阈值后，断路器打开
    - 超过恢复超时时间后，允许一次尝试
    - 如果尝试成功，断路器关闭；如果失败，重新打开
    """
    def decorator(func: Callable):
        breaker = CircuitBreaker(fail_max=fail_max, timeout_duration=timedelta(seconds=timeout_duration))
        
        @wraps(func)
        def wrapper(*args, **kwargs):
            if isinstance(breaker.state, CircuitOpenState):
                raise CircuitBreakerOpenError(f"Circuit breaker is open")
            return breaker.call(func, *args, **kwargs)
        return wrapper
    return decorator
```
