## 🎯 一、 产品愿景与核心逻辑 (Product Overview)
**Offer-Catcher** 是一款专为 AI Agent 开发求职者打造的“面经收集、宏观分析与模拟对练”智能体系统。
核心机制包含：
1. **多模态结构化**：利用 Vision LLM 提取面经图片，并进行意图分类熔断（拦截针对个人简历的提问）。
2. **混合 RAG**：Qdrant 向量库（单题粒度+上下文拼接） + 图数据库（宏观考频实体统计）。
3. **Multi-Agent 异步答疑**：通过 RabbitMQ 削峰填谷，由主系统将“客观八股题”分发给后台的 `Web_Search_Agent` 生成最新标准答案。
4. **自适应对练状态机**：通过 0/1/2 三档熟练度（Mastery Level）与打分 Agent (Scorer Agent) 形成闭环复习流。

## 🧩 三、 核心领域模型设计 (Domain Models & Schemas)
必须使用 `Pydantic` 在 `app/models/schemas.py` 和 `enums.py` 中严格定义。

### 1. 核心枚举 (Enums)
* **`QuestionType`**: `knowledge` (客观基础题), `project` (项目深挖题), `behavioral` (行为题), `scenario` (场景题)。
* **`MasteryLevel`**: `0` (完全不会/未复习), `1` (比较熟悉), `2` (已掌握)。

### 2. 系统数据总线 (Vision Extractor Output)
```python
class QuestionItem(BaseModel):
    question_id: str             # 必须为 MD5(company + question_text)
    question_text: str
    question_type: QuestionType  # LLM 判别: knowledge/project/behavioral/scenario
    requires_async_answer: bool  # 仅 knowledge/scenario 类型为 True
    core_entities: List[str]     # 提取的考察知识点实体
    mastery_level: MasteryLevel = MasteryLevel.LEVEL_0

class ExtractedInterview(BaseModel):
    source_type: str = "image"
    company: str                 # 需经过静态词典标准化
    position: str
    questions: List[QuestionItem]
```

### 3. 向量数据库 (Qdrant Schema)
* **Vector**: 采用上下文拼接 (Context Enrichment)。例如：`"公司：字节跳动 | 岗位：Agent应用开发 | 题目：qlora怎么优化显存？"`
* **Payload (Metadata)**: 必须包含 `company` (keyword), `position` (keyword), `mastery_level` (integer), `question_type` (keyword), `created_at` (datetime)。以便支持 Agent 的精确预过滤 (Pre-filtering)。
* **字段说明**:
  * `question_answer`: 生成的标准答案（异步生成后更新）
  * `created_at`: 创建时间戳，用于数据分析和时间排序

## ⚙️ 四、 核心业务流水线 (Core Workflows)

### 流水线 A：知识入库与异步答疑流 (Ingestion & Async Answer)
1. **入口**：`pipelines/ingestion.py` 接收图片路径或文本。
2. **提取与分类**：调用 `agents/vision_extractor.py` 输出符合 `ExtractedInterview` Schema 的数据。
3. **分发落库**：
   * 将每道题拆分为独立 Chunk 写入 Qdrant。
   * 筛选出 `requires_async_answer == True` 的题目，将**完整上下文（含公司岗位）**和题目打包发送至 RabbitMQ。对于 `project` 类问题直接熔断，跳过 MQ。
4. **异步消费**：`workers/answer_worker.py` 消费消息 -> 唤醒 `answer_specialist.py` -> 调用 `web_search.py` -> 生成标准答案 -> 基于 `question_id` (MD5) 执行 Qdrant 的 Upsert 更新答案。

### 流水线 B：检索与服务流 (Retrieval)
1. **入口**：`pipelines/retrieval.py` 接收自然语言 Query。
2. **向量检索**：使用 `tools/embedding.py` 生成查询向量，结合 Payload Filter（如 `{“mastery_level”: 0}`）进行混合检索。
3. **组装返回**：将检索到的题目（及后台异步生成的标准答案）格式化返回。

### 流水线 C：判分对练流 (Scoring State Machine) - (架构预留，供 Phase 2 实现)
1. **触发**：检测到用户带有上下文（引用某道题）的回答。
2. **执行**：唤醒 `agents/scorer.py`（预留），从 Qdrant 拉取该题的”标准答案”作为基准。对比用户输入，给出反馈。
3. **状态跃迁**：根据评价结果，Upsert 更新 Qdrant 中的 `mastery_level` (0 -> 1 或 1 -> 2)。
