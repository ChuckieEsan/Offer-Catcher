### 1.2 核心问题

| 问题 | 现状 | 影响 |
|------|------|------|
| **召回无筛选** | 检索到的 5 条全部注入 | 噪音记忆干扰对话 |
| **无相关性判断** | 不判断与当前查询相关性 | 话题切换后旧话题也注入 |
| **Memory Agent 只写不读** | Agent 只参与写入流程 | 召回流程无智能判断 |
| **硬编码阈值** | 固定 top_k=5, min_length=10 | 无法适应不同场景 |
| **无重要性权重** | 所有记忆平等对待 | 重要记忆可能被挤掉 |
| **无话题分离** | 检索时不区分话题 | 检索结果混杂不同话题 |

## 二、优化目标

### 2.1 设计原则

| 原则 | 说明 |
|------|------|
| **Agent 决策** | 让 LLM Agent 参与记忆筛选，而非硬编码规则 |
| **自适应权重** | 基于用户反馈动态调整，而非固定阈值 |
| **话题分离** | 话题匹配优先，减少噪音 |
| **分层存储** | STM/LTM 分层，重要记忆长期保留 |
| **延迟可控** | Agent 筛选不阻塞主流程 |

### 2.2 核心改进

| 改进点 | Before | After |
|--------|--------|-------|
| 记忆筛选 | 无筛选，全部注入 | Agent 精选，只注入相关记忆 |
| 召回流程 | 异步 fire-and-forget | 异步 + Agent 筛选 + 结果缓存 |
| 写入流程 | 硬编码阈值 | LLM 自判断 + 用户反馈驱动 |
| 记忆分层 | 单层平等 | STM/LTM 双层 + 自动升级 |
| 话题处理 | 无话题 | Agent 提取话题 + 话题匹配 |

## 四、Memory Write Agent 设计

### 4.1 Agent 目标

让 LLM 自决策，而非硬编码规则：
- 是否值得记住？
- 重要性评级？
- 涉及哪些话题？
- 应该放在 STM 还是 LTM？

### 4.2 提示词设计

```markdown
---
name: memory-write-agent
description: 从对话中提取有价值信息并写入记忆。使用 LLM 自决策而非硬编码规则。
---

<role>
你是一个记忆评估专家，负责从对话中提取有价值的信息并写入记忆系统。
你的核心任务是理解对话语义，自主判断记忆的价值和属性。
</role>

<task>
分析游标之后的新消息，自主判断并提取记忆内容。
</task>

<decision_framework>
你需要做出以下判断（基于语义理解，而非硬编码规则）：

### 判断 1：是否值得记住？

**应该记住的情况**：
- 用户表达了明确的偏好或期望
- 达成了具体的结论或方案
- 解决了问题或提供了可复用经验
- 深度讨论某个话题（多轮追问）

**不值得记住的情况**：
- 闲聊、问候、简单确认
- 一次性临时需求
- 模糊无意图的表达
- 简单问答无深度

### 判断 2：重要性评级

根据你的理解，给出重要性评级：

- **high**：用户偏好、重要结论、深度讨论结论
- **medium**：有价值的讨论、问题解决过程
- **low**：一般性对话、参考信息

### 判断 3：话题标签

提取对话涉及的核心话题：
- 技术名词（如 rabbitmq, kafka, lora）
- 领域概念（如 消息队列, 微调, 向量检索）
- 抽象主题（如 性能优化, 架构设计）

### 判断 4：记忆层级

- **long_term**：高重要性记忆，长期保留，不衰减
- **short_term**：中等重要性，可能随时间衰减
- **skip**：不值得记住，直接结束
</decision_framework>

<output_format>
请以 JSON 格式输出你的判断：

```json

    },
    "memory_content": {
        "summary": "简洁的会话摘要（20-50字）",
        "key_points": ["要点1", "要点2"]
    },
    "reasoning": "判断理由（简要说明为什么做出这些判断）"
}
```

```json

        "should_remember": false
    },

</output_format>

### 4.3 Agent 实现

```python
async def run_memory_write_agent(
    user_id: str,
    conversation_id: str,
    messages: list[BaseMessage],
) -> dict | None:
    """执行 Memory Write Agent
    
    流程：
    1. LLM 自判断是否值得记住
    2. 如果值得，继续判断重要性、话题、层级
    3. 写入数据库
    
    Args:
        user_id: 用户 ID
        conversation_id: 对话 ID
        messages: 对话消息列表
        
    Returns:
        写入的记忆实体，或 None（不值得记住）
    """
    # === 1. 获取游标后的新消息 ===
    new_messages = get_messages_since_cursor(messages, cursor_uuid)
    
    if not new_messages:
        return None
    
    # === 2. 构建上下文 ===
    context = {
        "new_messages": format_messages(new_messages),
        "current_preferences": repo.read_reference(user_id, "preferences") or "",
        "current_behaviors": repo.read_reference(user_id, "behaviors") or "",
        "conversation_id": conversation_id,
        "user_id": user_id,
        "cursor_uuid": new_messages[-1].id,
    }
    
    # === 3. 调用 Agent ===
    agent = create_memory_write_agent()
    input_message = HumanMessage(content=build_prompt_from_template(context))
    
    result = await agent.ainvoke({"messages": [input_message]})
    
    # === 4. 解析 Agent 输出 ===
    response_content = result["messages"][-1].content
    decision = parse_json_response(response_content)
    
    # === 5. 根据 Agent 判断处理 ===
    if not decision.get("should_remember", False):
        logger.info(f"Agent decided: not worth remembering. Reason: {decision.get('reasoning')}")
        update_cursor(user_id, conversation_id, context["cursor_uuid"])
        return None
    
    # === 6. 写入记忆 ===
    memory_content = decision.get("memory_content", {})
    summary = memory_content.get("summary", "")
    topics = decision.get("topics", [])
    
    # 重要性映射
    importance_map = {"high": 0.8, "medium": 0.5, "low": 0.3}
    importance_score = importance_map.get(decision.get("importance", "medium"), 0.5)
    
    # 层级映射
    layer_map = {"long_term": MemoryLayer.LTM, "short_term": MemoryLayer.STM}
    memory_layer = layer_map.get(decision.get("memory_layer", "short_term"), MemoryLayer.STM)
    
    # 计算 embedding
    embedding = embedding_adapter.embed(summary)
    
    # 创建记忆实体
    session_summary = SessionSummary.create(
        id=str(uuid.uuid4()),
        conversation_id=conversation_id,
        user_id=user_id,
        summary=summary,
        embedding=embedding,
        topics=topics,
        importance_score=importance_score,
        memory_layer=memory_layer,
    )
    
    # 写入数据库
    repo.create(session_summary)
    
    logger.info(f"Memory written: layer={memory_layer}, importance={importance_score}, topics={topics}")
    
    # === 7. 更新游标 ===
    update_cursor(user_id, conversation_id, context["cursor_uuid"])
    
    return session_summary
```

## 五、Memory Recall Agent 设计

### 5.1 Agent 目标

从候选池中精选相关记忆：
- 分析用户查询意图
- 逐条判断记忆相关性
- 决定是否注入
- 返回精选结果（而非全部）

### 5.2 提示词设计

```markdown
---
name: memory-recall-agent
description: 从候选记忆池中精选与当前查询相关的记忆。
---

<role>
你是一个记忆筛选专家，负责从候选记忆池中挑选与当前查询最相关的记忆。
你的目标是减少噪音，只注入真正有价值的记忆。
</role>

<task>
分析候选记忆池，判断每条记忆是否应该注入到当前对话。
</task>

<decision_framework>
对于每条候选记忆，你需要判断：

### 判断标准

**应该注入的情况**：
- 与当前查询话题直接相关
- 包含解决当前问题所需的信息
- 是用户的偏好表达（与当前场景相关）
- 是之前同一话题的讨论结论

**不应该注入的情况**：
- 与当前查询话题不相关
- 是之前其他话题的讨论（已切换话题）
- 信息过时或已被后续讨论覆盖
- 与当前问题解决无关

### 特殊考虑

- **LTM 记忆**：用户偏好通常应该注入（除非明显不相关）
- **STM 记忆**：严格按话题相关性判断
- **同一 conversation 的记忆**：如果是当前对话的记忆，优先注入
</decision_framework>

<output_format>
请以 JSON 格式输出你的判断：

```json
{
    "selected_memories": [
        {
            "memory_id": "xxx",
            "inject": true,
            "inject_order": 1,
            "reason": "与查询直接相关，包含解决方案"
        },
        {
            "memory_id": "yyy",
            "inject": false,
            "reason": "话题不相关，是之前的其他讨论"
        }
    ],
    "analysis": {
        "query_intent": "用户想要了解 RAG 召回阈值设置",
        "query_topics": ["RAG", "召回阈值"],
        "relevant_count": 2,
        "skipped_count": 8
    }
}
```

`inject_order` 表示注入顺序（1 最优先），最多选择 5 条注入。
</output_format>

<examples>

**候选池**：

3. [STM] 讨论了 lora 的原理（2天前）
4. [STM] 讨论了 qlora 的显存优化（昨天）
5. [STM] 今天天气不错（闲聊）

### 5.3 Agent 实现

```python
async def run_memory_recall_agent(
    user_id: str,
    query: str,
    candidate_memories: list[SessionSummary],
    active_topics: set[str],
) -> list[SessionSummary]:
    """执行 Memory Recall Agent
    
    流程：
    1. LLM 分析候选池
    2. 判断每条记忆是否应该注入
    3. 返回精选结果
    
    Args:
        user_id: 用户 ID
        query: 用户查询
        candidate_memories: 候选记忆池
        active_topics: 当前活跃话题
        
    Returns:
        精选的记忆列表（最多 5 条）
    """
    if not candidate_memories:
        return []
    
    # === 1. 构建上下文 ===
    context = {
        "query": query,
        "candidate_memories": format_candidate_memories(candidate_memories),
        "active_topics": list(active_topics),
    }
    
    # === 2. 调用 Agent ===
    agent = create_memory_recall_agent()
    input_message = HumanMessage(content=build_prompt_from_template(context))
    
    result = await agent.ainvoke({"messages": [input_message]})
    
    # === 3. 解析 Agent 输出 ===
    response_content = result["messages"][-1].content
    decision = parse_json_response(response_content)
    
    # === 4. 筛选记忆 ===
    selected = decision.get("selected_memories", [])
    
    # 按 inject_order 排序，只取 inject=true 的
    inject_decisions = [
        (s["memory_id"], s["inject_order"])
        for s in selected
        if s.get("inject", False)
    ]
    
    # 按 order 排序
    inject_decisions.sort(key=lambda x: x[1])
    
    # 映射到记忆实体
    memory_map = {m.id: m for m in candidate_memories}
    
    selected_memories = [
        memory_map[mid]
        for mid, _ in inject_decisions[:5]  # 最多 5 条
        if mid in memory_map
    ]
    
    # === 5. 更新访问计数 ===
    for memory in selected_memories:
        memory.access_count += 1
        memory.last_accessed = datetime.now()
        repo.update(memory)
    
    logger.info(f"Memory recall: {len(candidate_memories)} candidates → {len(selected_memories)} selected")
    
    return selected_memories


def format_candidate_memories(memories: list[SessionSummary]) -> str:
    """格式化候选记忆池
    
    输出格式：
    1. [LTM] 用户偏好简洁回答 (importance=0.9)
    2. [STM] 讨论了 rabbitmq vs kafka (importance=0.7, topics=[rabbitmq, kafka])
    """
    lines = []
    for i, m in enumerate(memories, 1):
        layer = "LTM" if m.memory_layer == MemoryLayer.LTM else "STM"
        topics_str = ", ".join(m.topics) if m.topics else "无"
        lines.append(
            f"{i}. [{layer}] {m.summary[:50]} "
            f"(importance={m.importance_score}, topics=[{topics_str}], created={m.created_at.strftime('%Y-%m-%d')})"
        )
    
    return "\n".join(lines)
```

### 6.1 三阶段召回

```python
class MemoryRecallPipeline:
    """记忆召回流程
    
    三阶段：
    1. 候选池检索（向量 + 话题）
    2. Agent 精选（相关性判断）
    3. 格式化注入
    """
    
    async def retrieve(
        self,
        user_id: str,
        query: str,
        active_topics: set[str],
    ) -> str:
        """完整召回流程
        
        Args:
            user_id: 用户 ID
            query: 用户查询
            active_topics: 当前活跃话题
            
        Returns:
            格式化的 memory_context
        """
        # === 第一阶段：候选池检索 ===
        candidates = await self._retrieve_candidates(user_id, query, active_topics)
        
        # === 第二阶段：Agent 精选 ===
        selected = await run_memory_recall_agent(user_id, query, candidates, active_topics)
        
        # === 第三阶段：格式化 ===
        memory_context = self._format_memory_context(selected)
        
        return memory_context
    
    async def _retrieve_candidates(
        self,
        user_id: str,
        query: str,
        active_topics: set[str],
    ) -> list[SessionSummary]:
        """第一阶段：候选池检索
        
        策略：
        - LTM 全量召回（重要记忆）
        - STM 话题优先召回
        - STM 语义补充召回
        - 去重合并
        """
        query_embedding = embedding_adapter.embed(query)
        query_topics = set(extract_topics(query))
        
        # 1. LTM 全量召回
        ltm_memories = repo.search_by_embedding(
            user_id=user_id,
            query_embedding=query_embedding,
            memory_layer=MemoryLayer.LTM,
            top_k=15,  # LTM 容量大
        )
        
        # 2. STM 话题优先召回
        stm_topic_memories = repo.search_by_embedding_with_topics(
            user_id=user_id,
            query_embedding=query_embedding,
            topics=query_topics.union(active_topics),
            memory_layer=MemoryLayer.STM,
            top_k=10,
        )
        
        # 3. STM 语义补充召回
        stm_semantic_memories = repo.search_by_embedding(
            user_id=user_id,
            query_embedding=query_embedding,
            memory_layer=MemoryLayer.STM,
            exclude_ids=[m.id for m in stm_topic_memories],
            top_k=5,
        )
        
        # 4. 去重合并
        all_memories = ltm_memories + stm_topic_memories + stm_semantic_memories
        unique_memories = self._deduplicate(all_memories)
        
        logger.info(f"Candidates retrieved: LTM={len(ltm_memories)}, STM_topic={len(stm_topic_memories)}, STM_semantic={len(stm_semantic_memories)}")
        
        return unique_memories
    
    def _deduplicate(self, memories: list[SessionSummary]) -> list[SessionSummary]:
        """按 conversation_id 去重"""
        seen = {}
        for m in memories:
            if m.conversation_id not in seen:
                seen[m.conversation_id] = m
            else:
                # 保留重要性更高的
                if m.importance_score > seen[m.conversation_id].importance_score:
                    seen[m.conversation_id] = m
        
        return list(seen.values())
    
    def _format_memory_context(self, memories: list[SessionSummary]) -> str:
        """格式化 memory_context
        
        格式：
        - LTM 优先，标记 [重要]
        - STM 补充
        """
        if not memories:
            return ""
        
        sections = []
        
        # LTM 先
        for m in memories:
            if m.memory_layer == MemoryLayer.LTM:
                topics_str = ", ".join(m.topics) if m.topics else "通用"
                sections.append(f"### [重要] {m.summary[:50]}\n话题：{topics_str}\n详情：{m.summary}")
        
        # STM 后
        for m in memories:
            if m.memory_layer == MemoryLayer.STM:
                topics_str = ", ".join(m.topics) if m.topics else "通用"
                sections.append(f"### {m.summary[:50]}\n话题：{topics_str}\n详情：{m.summary}")
        
        return "\n\n".join(sections)
```

## 七、延迟优化策略

### 7.1 问题：Agent 筛选增加延迟

Agent 筛选需要额外 LLM 调用，可能增加 500ms-2000ms 延迟。

### 7.2 优化方案

#### 方案 A：异步预召回 + 缓存

```python
class CachedMemoryRecall:
    """缓存式召回
    
    策略：
    1. 用户上一轮查询结束时，异步预召回下一轮可能需要的记忆
    2. 缓存结果，下轮直接使用
    3. 如果缓存失效，降级到快速召回（无 Agent）
    """
    
    def __init__(self):
        self.cache: dict[str, MemoryRecallResult] = {}
        self.cache_ttl = 60  # 缓存 60 秒
    
    async def pre_retrieve(self, user_id: str, last_query: str) -> None:
        """异步预召回（对话结束后触发）"""
        # 触发异步任务
        asyncio.create_task(self._do_pre_retrieve(user_id, last_query))
    
    async def _do_pre_retrieve(self, user_id: str, query: str) -> None:
        """执行预召回"""
        result = await recall_pipeline.retrieve(user_id, query)
        
        # 缓存结果
        cache_key = f"{user_id}:{hash(query)}"
        self.cache[cache_key] = MemoryRecallResult(
            result=result,
            timestamp=datetime.now(),
        )
    
    async def retrieve_cached(self, user_id: str, query: str) -> str:
        """获取缓存或降级"""
        cache_key = f"{user_id}:{hash(query)}"
        
        # 1. 尝试缓存
        if cache_key in self.cache:
            cached = self.cache[cache_key]
            if (datetime.now() - cached.timestamp).seconds < self.cache_ttl:
                logger.info(f"Memory recall: using cached result")
                return cached.result
        
        # 2. 缓存失效 → 降级快速召回
        logger.info(f"Memory recall: cache miss, using fast fallback")
        return await self._fast_recall(user_id, query)
    
    async def _fast_recall(self, user_id: str, query: str) -> str:
        """快速召回（无 Agent，仅向量检索）"""
        query_embedding = embedding_adapter.embed(query)
        
        # 只做向量检索，无 Agent 筛选
        memories = repo.search_by_embedding(
            user_id=user_id,
            query_embedding=query_embedding,
            top_k=5,
        )
        
        return format_memories(memories)
```

```python
async def parallel_recall_with_agent(
    user_id: str,
    query: str,
) -> str:
    """并行召回
    
    策略：
    1. 并行执行：向量检索 + Agent 筛选
    2. Agent 筛选在后台运行
    3. 如果 Agent 完成，使用精选结果
    4. 如果 Agent 未完成，使用向量检索结果（降级）
    """
    # 1. 启动向量检索（快）
    vector_task = asyncio.create_task(
        fast_vector_retrieve(user_id, query)
    )
    
    # 2. 启动 Agent 筛选（慢）
    agent_task = asyncio.create_task(
        agent_select_retrieve(user_id, query)
    )
    
    # 3. 等待向量检索完成（必须）
    vector_result = await vector_task  # ~100ms
    
    # 4. 检查 Agent 是否完成
    try:
        agent_result = await asyncio.wait_for(
            agent_task, timeout=0.5  # 最多等 500ms
        )
        return agent_result  # Agent 完成，使用精选结果
    except asyncio.TimeoutError:
        return vector_result  # Agent 未完成，降级使用向量结果
```

### 7.3 推荐方案

| 场景 | 推荐 | 理由 |
|------|------|------|
| **高延迟容忍** | 同步 Agent 筛选 | 精选质量最高 |
| **中等延迟容忍** | 并行召回 + Agent | Agent 超时自动降级 |
| **低延迟要求** | 异步预召回缓存 | 预计算，延迟最低 |

    衰减策略：
    - STM 衰减快（基于访问计数）

class FeedbackCollector:
    """用户反馈收集
    
    反馈来源：
    1. 显式反馈：点赞/踩按钮
    2. 隐式反馈：追问、感谢、抱怨
    """
    
    # 隐式反馈信号
    POSITIVE_SIGNALS = ["谢谢", "好的", "明白了", "有用", "可行"]
    NEGATIVE_SIGNALS = ["不对", "错了", "没有帮助", "太长了", "不好"]
    
    def detect_implicit_feedback(self, user_message: str) -> str | None:
        """检测隐式反馈
        
        Returns:
            "positive" / "negative" / None
        """
        msg_lower = user_message.lower()
        
        for signal in self.POSITIVE_SIGNALS:
            if signal in msg_lower:
                return "positive"
        
        for signal in self.NEGATIVE_SIGNALS:
            if signal in msg_lower:
                return "negative"
        
        return None
    
    async def process_feedback(
        self,
        memory_id: str,
        feedback_type: str,
        user_id: str,
    ) -> None:
        """处理反馈
        
        反馈影响：
        - positive → importance +0.1, feedback_score +1
        - negative → importance -0.2, feedback_score -1
        """
        memory = repo.get_by_id(memory_id)
        
        if feedback_type == "positive":
            memory.importance_score = min(memory.importance_score + 0.1, 1.0)
            memory.feedback_score += 1
            
            # 多次正向 → 升级
            if memory.feedback_score >= 3 and memory.memory_layer == MemoryLayer.STM:
                await upgrade_memory(memory)
        
        elif feedback_type == "negative":
            memory.importance_score = max(memory.importance_score - 0.2, 0.0)
            memory.feedback_score -= 1
            
            # 多次负向 → 删除
            if memory.feedback_score <= -2:
                memory.marked_for_deletion = True
        
        repo.update(memory)
```

## 十三、风险与缓解

### 13.1 潜在风险

| 风险 | 说明 | 缓解措施 |
|------|------|----------|
| Agent 增加延迟 | 500ms-2000ms | 缓存 + 并行 + 降级 |
| Agent 判断偏差 | 可能误判 | 用户反馈纠正 |
| 迁移成本 | 数据库变更 | 分阶段实施 |
| JSON 解析失败 | Agent 输出格式问题 | 重试 + 降级 |

### 13.2 降级方案

```python
async def safe_recall_with_fallback(user_id: str, query: str) -> str:
    """安全召回（带降级）"""
    try:
        # 尝试 Agent 筛选
        return await agent_select_retrieve(user_id, query)
    except Exception as e:
        logger.warning(f"Agent recall failed: {e}, using vector fallback")
        # 降级到纯向量检索
        return await fast_vector_retrieve(user_id, query)


async def safe_write_with_fallback(messages: list) -> None:
    """安全写入（带降级）"""
    try:
        # 尝试 Agent 判断
        await agent_write_memory(messages)
    except Exception as e:
        logger.warning(f"Agent write failed: {e}, using simple write")
        # 降级到简单写入（写入 STM，中等重要性）
        await simple_write_memory(messages)
```
