> 版本：v2.0
> 日期：2026-04-22
> 状态：设计阶段

## 一、现有问题分析

### 1.1 核心问题总结

当前记忆模块存在以下关键缺陷：

| 问题类别 | 具体问题 | 影响 |
|----------|----------|------|
| **权重缺失** | 所有记忆平等对待，无重要性评分 | 高价值讨论可能被低价值闲聊挤掉 |
| **话题未分离** | 所有话题的摘要混在一起存储 | 检索时引入不相关话题噪音 |
| **单层架构** | 无 STM/LTM 分层，所有记忆同等衰减 | 深度讨论无法长期保留 |
| **召回策略单一** | 仅依赖向量相似度检索 | 无法区分当前话题和历史话题 |
| **无升级机制** | 记忆无法从短期升级到长期 | 重要讨论永远停留在低优先级 |
| **被动遗忘** | 依赖容量裁剪被动删除 | 旧话题永远占用空间 |

### 1.2 边界场景分析

#### 场景 A：深度讨论被遗忘

```
第1-20轮：深度讨论 rabbitmq vs kafka（得出重要结论）
第21轮：切换到 lora vs qlora
第40轮：回到 rabbitmq/kafka 话题

问题：rabbitmq/kafka 讨论已被裁剪，Agent 无法回忆之前的结论
```

#### 场景 B：话题切换噪音

```
当前讨论：lora 微调技术
检索时返回：lora 相关 + rabbitmq/kafka 相关（因为之前讨论过）

问题：不相关话题干扰当前对话焦点
```

#### 场景 C：重要记忆被挤掉

```
用户偏好："我喜欢简洁回答"（重要，应长期记住）
闲聊摘要："今天天气不错"（不重要）
容量限制导致：偏好被闲聊挤掉

问题：缺乏权重导致高价值内容丢失
```

## 二、优化目标

### 2.1 核心目标

| 目标 | 指标 | 说明 |
|------|------|------|
| 重要记忆保留 | LTM 记忆不被裁剪 | 深度讨论、用户偏好长期可召回 |
| 话题隔离 | 检索命中率 > 80% | 只召回相关话题记忆 |
| 自适应遗忘 | STM 衰减率 > LTM | 非重要记忆快速清理 |
| 优先召回 | 话题匹配 + 语义相似 | 多维度召回策略 |

### 2.2 设计原则

1. **重要性驱动**：高价值记忆优先保留
2. **话题分离**：不同话题记忆独立管理
3. **层级升级**：STM → LTM 自动迁移
4. **主动遗忘**：不相关话题主动清理
5. **混合召回**：话题匹配 + 语义相似 + 时间衰减

### 3.2 记忆数据模型

```python
class MemoryLayer(str, Enum):
    """记忆层级"""
    STM = "short_term"   # 短期记忆，快速衰减
    LTM = "long_term"    # 长期记忆，慢速衰减


class SessionSummary(BaseModel):
    """会话摘要实体（优化版）"""
    
    id: str = Field(description="摘要唯一标识")
    conversation_id: str = Field(description="关联对话 ID")
    user_id: str = Field(description="用户 ID")
    summary: str = Field(description="会话摘要内容")
    embedding: Optional[list[float]] = Field(description="语义向量")
    
    # === 新增字段 ===
    
    # 重要性评分
    importance_score: float = Field(
        default=0.5,
        ge=0.0,
        le=1.0,
        description="重要性评分，0-1，越高越重要"
    )
    
    # 话题标签
    topics: list[str] = Field(
        default_factory=list,
        description="话题标签，如 ['rabbitmq', 'kafka', '消息队列']"
    )
    
    # 记忆层级
    memory_layer: MemoryLayer = Field(
        default=MemoryLayer.STM,
        description="STM 或 LTM"
    )
    
    # 记忆强度（访问次数）
    access_count: int = Field(
        default=0,
        description="被召回次数，多次访问增强记忆强度"
    )
    
    # 时间戳
    created_at: datetime = Field(description="创建时间")
    last_accessed: datetime = Field(description="最后访问时间")
    last_decay: datetime = Field(description="上次衰减时间")
    
    # 衰减因子
    decay_factor: float = Field(
        default=1.0,
        ge=0.0,
        le=1.0,
        description="衰减因子，随时间递减"
    )
    
    @classmethod
    def create(
        cls,
        id: str,
        conversation_id: str,
        user_id: str,
        summary: str,
        embedding: list[float],
        topics: list[str],
        importance_score: float = 0.5,
    ) -> "SessionSummary":
        """创建会话摘要"""
        now = datetime.now()
        
        # 根据重要性决定初始层级
        initial_layer = (
            MemoryLayer.LTM if importance_score >= 0.7
            else MemoryLayer.STM
        )
        
        return cls(
            id=id,
            conversation_id=conversation_id,
            user_id=user_id,
            summary=summary,
            embedding=embedding,
            topics=topics,
            importance_score=importance_score,
            memory_layer=initial_layer,
            access_count=0,
            created_at=now,
            last_accessed=now,
            last_decay=now,
            decay_factor=1.0,
        )
    
    def calculate_retrieval_score(self, query_topics: set[str]) -> float:
        """计算检索得分
        
        多维度综合评分：
        - 话题匹配度
        - 语义相似度（外部计算）
        - 重要性评分
        - 记忆强度（访问次数）
        - 衰减因子
        
        Returns:
            综合检索得分 0-1
        """
        # 1. 话题匹配度
        topic_overlap = len(set(self.topics).intersection(query_topics))
        topic_score = min(topic_overlap / max(len(query_topics), 1), 1.0)
        
        # 2. 重要性评分
        importance_score = self.importance_score
        
        # 3. 记忆强度（访问次数归一化）
        strength_score = min(self.access_count / 10, 1.0)
        
        # 4. 衰减因子
        decay_score = self.decay_factor
        
        # 综合评分（加权平均）
        # LTM 记忆：话题匹配权重低（始终召回）
        # STM 记忆：话题匹配权重高（精准召回）
        if self.memory_layer == MemoryLayer.LTM:
            weights = {
                "topic": 0.1,
                "importance": 0.4,
                "strength": 0.3,
                "decay": 0.2,
            }
        else:
            weights = {
                "topic": 0.4,
                "importance": 0.2,
                "strength": 0.2,
                "decay": 0.2,
            }
        
        total = (
            weights["topic"] * topic_score +
            weights["importance"] * importance_score +
            weights["strength"] * strength_score +
            weights["decay"] * decay_score
        )
        
        return total
```

## 四、核心机制设计

### 4.1 重要性评分机制

#### 评分维度

```python
def calculate_importance_score(
    summary: str,
    conversation_context: dict,
) -> float:
    """计算记忆重要性评分
    
    评分维度：
    1. 信息密度（技术关键词数量）
    2. 问题解决（是否给出具体方案/结论）
    3. 用户反馈（正向确认/感谢）
    4. 话题深度（讨论轮次）
    5. 可复用性（通用经验 vs 特定场景）
    
    Args:
        summary: 会话摘要文本
        conversation_context: 对话上下文信息
        
    Returns:
        重要性评分 0-1
    """
    score = 0.0
    
    # === 维度 1：信息密度 ===
    tech_keywords = extract_technical_keywords(summary)
    # 技术关键词越多，信息密度越高
    density_score = min(len(tech_keywords) * 0.05, 0.25)
    score += density_score
    
    # === 维度 2：问题解决 ===
    solution_patterns = [
        "方案是",
        "解决方案",
        "建议",
        "步骤如下",
        "结论是",
        "可以这样做",
    ]
    has_solution = any(pattern in summary for pattern in solution_patterns)
    if has_solution:
        score += 0.25
    
    # === 维度 3：用户反馈 ===
    user_messages = conversation_context.get("user_messages", [])
    positive_patterns = ["好的", "明白了", "谢谢", "有用", "可行", "这个方案"]
    has_positive_feedback = any(
        any(pattern in msg for pattern in positive_patterns)
        for msg in user_messages[-3:]  # 最近 3 条用户消息
    )
    if has_positive_feedback:
        score += 0.15
    
    # === 维度 4：话题深度 ===
    turn_count = conversation_context.get("turn_count", 1)
    # 讨论轮次越多，深度越高
    depth_score = min(turn_count * 0.02, 0.15)
    score += depth_score
    
    # === 维度 5：可复用性 ===
    # 判断是否是通用经验（而非特定场景）
    reusable_patterns = [
        "一般",
        "通常",
        "常见",
        "推荐",
        "最佳实践",
    ]
    has_reusable = any(pattern in summary for pattern in reusable_patterns)
    if has_reusable:
        score += 0.10
    
    # === 维度 6：偏好表达 ===
    preference_patterns = [
        "我喜欢",
        "我偏好",
        "我希望",
        "我不喜欢",
        "不要",
    ]
    has_preference = any(pattern in summary for pattern in preference_patterns)
    if has_preference:
        score += 0.25  # 偏好表达权重最高
    
    return min(score, 1.0)
```

#### 评分阈值

| 评分范围 | 层级 | 说明 |
|----------|------|------|
| 0.7 - 1.0 | LTM | 高重要性，长期保留 |
| 0.5 - 0.7 | STM（候选 LTM） | 中等重要性，可能升级 |
| 0.3 - 0.5 | STM | 一般重要性，正常衰减 |
| 0.0 - 0.3 | 不存储 | 低价值，不写入记忆 |

### 4.2 话题提取与分离

#### 话题提取

```python
def extract_topics(text: str) -> list[str]:
    """从文本中提取话题标签
    
    策略：
    1. 技术名词提取（NLP）
    2. 领域关键词匹配
    3. LLM 辅助提取（可选）
    
    Args:
        text: 输入文本
        
    Returns:
        话题标签列表
    """
    topics = []
    
    # === 策略 1：技术名词提取 ===
    # 使用预定义的技术关键词库
    tech_keywords_db = {
        # 消息队列
        "rabbitmq", "kafka", "activemq", "rocketmq", "消息队列", "mq",
        # AI/ML
        "lora", "qlora", "rag", "embedding", "vector", "llm", "gpt",
        "transformer", "attention", "fine-tuning", "微调",
        # 数据库
        "postgres", "mysql", "redis", "mongodb", "qdrant", "neo4j",
        # 编程语言
        "python", "java", "go", "rust", "typescript",
        # 架构
        "ddd", "微服务", "monolith", "api", "grpc", "rest",
    }
    
    text_lower = text.lower()
    for keyword in tech_keywords_db:
        if keyword in text_lower:
            topics.append(keyword)
    
    # === 策略 2：话题归一化 ===
    # 将相似话题归一化
    topic_groups = {
        "rabbitmq_kafka": ["rabbitmq", "kafka", "activemq", "rocketmq", "消息队列", "mq"],
        "lora_qlora": ["lora", "qlora", "微调", "fine-tuning"],
        "vector_db": ["qdrant", "milvus", "pinecone", "向量数据库"],
    }
    
    normalized_topics = []
    for topic in topics:
        for group_name, group_keywords in topic_groups.items():
            if topic in group_keywords:
                if group_name not in normalized_topics:
                    normalized_topics.append(group_name)
                break
        else:
            normalized_topics.append(topic)
    
    return normalized_topics


def detect_topic_switch(
    current_topics: set[str],
    new_query: str,
) -> tuple[bool, set[str]]:
    """检测话题切换
    
    Args:
        current_topics: 当前活跃话题集合
        new_query: 新的用户查询
        
    Returns:
        (是否切换, 新话题集合)
    """
    new_topics = set(extract_topics(new_query))
    
    # 完全不同的话题 = 切换
    if not current_topics.intersection(new_topics):
        return True, new_topics
    
    # 部分重叠 = 延续
    return False, current_topics.union(new_topics)
```

### 4.3 STM → LTM 升级机制

```python
class MemoryUpgradeService:
    """记忆升级服务
    
    负责 STM 记忆迁移到 LTM
    """
    
    # 升级阈值
    IMPORTANCE_THRESHOLD = 0.7
    ACCESS_COUNT_THRESHOLD = 3
    DISCUSSION_DEPTH_THRESHOLD = 10  # 讨论轮次
    
    def check_upgrade_conditions(self, memory: SessionSummary) -> bool:
        """检查是否满足升级条件
        
        升级条件（满足任一）：
        1. importance_score >= 0.7
        2. access_count >= 3（被多次召回）
        3. 同话题讨论轮次 >= 10
        
        Args:
            memory: 会话摘要
            
        Returns:
            是否满足升级条件
        """
        if memory.memory_layer == MemoryLayer.LTM:
            return False  # 已是 LTM
        
        # 条件 1：高重要性
        if memory.importance_score >= self.IMPORTANCE_THRESHOLD:
            return True
        
        # 条件 2：多次访问
        if memory.access_count >= self.ACCESS_COUNT_THRESHOLD:
            return True
        
        # 条件 3：深度讨论（需要额外统计）
        # 这里假设有话题深度统计服务
        # topic_depth = get_topic_depth(memory.topics)
        # if topic_depth >= self.DISCUSSION_DEPTH_THRESHOLD:
        #     return True
        
        return False
    
    def upgrade_to_ltm(self, memory_id: str) -> SessionSummary:
        """执行升级
        
        Args:
            memory_id: 记忆 ID
            
        Returns:
            升级后的记忆
        """
        memory = self.repo.get_by_id(memory_id)
        
        if not self.check_upgrade_conditions(memory):
            return memory
        
        # 执行升级
        memory.memory_layer = MemoryLayer.LTM
        memory.importance_score = max(memory.importance_score, 0.7)  # 确保重要性
        memory.decay_factor = 1.0  # 重置衰减
        
        self.repo.update(memory)
        
        logger.info(f"Memory upgraded to LTM: {memory_id}, score={memory.importance_score}")
        
        return memory
    
    def batch_upgrade_check(self, user_id: str) -> list[str]:
        """批量检查并升级
        
        定期扫描用户 STM 记忆，检查是否满足升级条件
        
        Args:
            user_id: 用户 ID
            
        Returns:
            升级的记忆 ID 列表
        """
        stm_memories = self.repo.get_stm_memories(user_id)
        upgraded_ids = []
        
        for memory in stm_memories:
            if self.check_upgrade_conditions(memory):
                self.upgrade_to_ltm(memory.id)
                upgraded_ids.append(memory.id)
        
        return upgraded_ids
```

### 4.4 衰减与遗忘机制

```python
class MemoryDecayService:
    """记忆衰减服务
    
    实现时间衰减和主动遗忘
    """
    
    # 衰减参数
    STM_DECAY_RATE = 0.3   # STM 快衰减
    LTM_DECAY_RATE = 0.05  # LTM 衰减
    
    # 衰减周期（秒）
    DECAY_INTERVAL = 3600 * 24  # 每天衰减一次
    
    # 遗忘阈值
    FORGET_THRESHOLD = 0.1  # decay_factor < 0.1 时删除
    
    def apply_decay(self, memory: SessionSummary) -> SessionSummary:
        """应用时间衰减
        
        衰减公式：
        decay_factor = decay_factor * exp(-λ * time_elapsed)
        
        Args:
            memory: 会话摘要
            
        Returns:
            衰减后的记忆
        """
        now = datetime.now()
        time_elapsed = (now - memory.last_decay).total_seconds() / self.DECAY_INTERVAL
        
        # 选择衰减速率
        decay_rate = (
            self.LTM_DECAY_RATE if memory.memory_layer == MemoryLayer.LTM
            else self.STM_DECAY_RATE
        )
        
        # 应用衰减
        memory.decay_factor *= math.exp(-decay_rate * time_elapsed)
        memory.last_decay = now
        
        # 记忆访问会增强衰减因子
        if memory.last_accessed > memory.last_decay:
            # 最近被访问过，衰减因子恢复
            memory.decay_factor = min(memory.decay_factor + 0.1, 1.0)
        
        return memory
    
    def check_forget(self, memory: SessionSummary) -> bool:
        """检查是否应该遗忘
        
        遗忘条件：
        1. STM 记忆：decay_factor < FORGET_THRESHOLD
        2. LTM 记忆：不遗忘（长期保留）
        
        Args:
            memory: 会话摘要
            
        Returns:
            是否应该遗忘
        """
        if memory.memory_layer == MemoryLayer.LTM:
            return False  # LTM 不主动遗忘
        
        return memory.decay_factor < self.FORGET_THRESHOLD
    
    def batch_decay_and_forget(self, user_id: str) -> tuple[int, int]:
        """批量衰减和遗忘
        
        Args:
            user_id: 用户 ID
            
        Returns:
            (衰减数量, 遗忘数量)
        """
        memories = self.repo.get_all_memories(user_id)
        decayed_count = 0
        forgotten_count = 0
        
        for memory in memories:
            # 应用衰减
            memory = self.apply_decay(memory)
            decayed_count += 1
            
            # 检查遗忘
            if self.check_forget(memory):
                self.repo.delete(memory.id)
                forgotten_count += 1
                logger.info(f"Memory forgotten: {memory.id}, decay={memory.decay_factor}")
            else:
                self.repo.update(memory)
        
        return decayed_count, forgotten_count
    
    def topic_switch_cleanup(
        self,
        user_id: str,
        new_topics: set[str],
    ) -> int:
        """话题切换时的记忆清理
        
        清理不相关话题的 STM 记忆（LTM 保留）
        
        Args:
            user_id: 用户 ID
            new_topics: 新话题集合
            
        Returns:
            清理的记忆数量
        """
        stm_memories = self.repo.get_stm_memories(user_id)
        cleaned_count = 0
        
        for memory in stm_memories:
            memory_topics = set(memory.topics)
            
            # 不相关话题 → 加速衰减
            if not memory_topics.intersection(new_topics):
                memory.decay_factor *= 0.5  # 加速衰减
                self.repo.update(memory)
                cleaned_count += 1
                logger.debug(f"Accelerated decay for unrelated topic: {memory.id}")
        
        return cleaned_count
```

### 4.5 混合召回策略

```python
class MemoryRetrievalService:
    """记忆召回服务
    
    实现多维度混合召回
    """
    
    # 召回参数
    LTM_TOP_K = 10        # LTM 召回数量
    STM_TOP_K = 5         # STM 召回数量
    MIN_RETRIEVAL_SCORE = 0.3  # 最小召回得分
    
    async def retrieve(
        self,
        user_id: str,
        query: str,
        active_topics: set[str],
    ) -> list[SessionSummary]:
        """混合召回
        
        召回策略：
        1. LTM 全量召回（不依赖话题匹配）
        2. STM 话题优先召回
        3. STM 语义补充召回
        4. 多维度排序
        
        Args:
            user_id: 用户 ID
            query: 用户查询
            active_topics: 当前活跃话题
            
        Returns:
            召回的记忆列表
        """
        query_embedding = self.embedding_adapter.embed(query)
        query_topics = set(extract_topics(query))
        
        # === 1. LTM 全量召回 ===
        ltm_memories = self.repo.get_ltm_memories(user_id, limit=self.LTM_TOP_K)
        
        # LTM 记忆更新访问计数
        for memory in ltm_memories:
            memory.access_count += 1
            memory.last_accessed = datetime.now()
            self.repo.update(memory)
        
        # === 2. STM 话题优先召回 ===
        stm_topic_matched = self.repo.search_stm_by_topics(
            user_id=user_id,
            topics=query_topics.union(active_topics),
            limit=self.STM_TOP_K,
        )
        
        # === 3. STM 语义补充召回 ===
        stm_semantic = self.repo.search_stm_by_embedding(
            user_id=user_id,
            query_embedding=query_embedding,
            exclude_ids=[m.id for m in stm_topic_matched],
            limit=self.STM_TOP_K,
        )
        
        # === 4. 合并去重 ===
        all_memories = ltm_memories + stm_topic_matched + stm_semantic
        unique_memories = self._deduplicate(all_memories)
        
        # === 5. 多维度排序 ===
        scored_memories = [
            (memory, memory.calculate_retrieval_score(query_topics))
            for memory in unique_memories
        ]
        
        # 过滤低得分记忆
        filtered_memories = [
            memory for memory, score in scored_memories
            if score >= self.MIN_RETRIEVAL_SCORE
        ]
        
        # 按得分排序
        sorted_memories = sorted(
            filtered_memories,
            key=lambda m: m.calculate_retrieval_score(query_topics),
            reverse=True,
        )
        
        return sorted_memories
    
    def _deduplicate(self, memories: list[SessionSummary]) -> list[SessionSummary]:
        """去重
        
        按 conversation_id 去重，保留得分最高的
        """
        seen = {}
        for memory in memories:
            if memory.conversation_id not in seen:
                seen[memory.conversation_id] = memory
            else:
                # 保留重要性更高的
                if memory.importance_score > seen[memory.conversation_id].importance_score:
                    seen[memory.conversation_id] = memory
        
        return list(seen.values())
    
    def format_for_context(
        self,
        memories: list[SessionSummary],
        max_size: int = 20 * 1024,
    ) -> str:
        """格式化为 memory_context
        
        Args:
            memories: 召回的记忆列表
            max_size: 最大容量
            
        Returns:
            memory_context 字符串
        """
        sections = []
        total_size = 0
        
        # LTM 记忆优先（标记为 [重要]）
        ltm_memories = [m for m in memories if m.memory_layer == MemoryLayer.LTM]
        stm_memories = [m for m in memories if m.memory_layer == MemoryLayer.STM]
        
        # 先添加 LTM
        for memory in ltm_memories:
            section = self._format_memory(memory, is_important=True)
            section_size = len(section.encode("utf-8"))
            
            if total_size + section_size > max_size:
                break
            
            sections.append(section)
            total_size += section_size
        
        # 再添加 STM
        for memory in stm_memories:
            section = self._format_memory(memory, is_important=False)
            section_size = len(section.encode("utf-8"))
            
            if total_size + section_size > max_size:
                break
            
            sections.append(section)
            total_size += section_size
        
        return "\n\n".join(sections)
    
    def _format_memory(
        self,
        memory: SessionSummary,
        is_important: bool,
    ) -> str:
        """格式化单个记忆"""
        topics_str = ", ".join(memory.topics) if memory.topics else "通用"
        importance_marker = "[重要]" if is_important else ""
        
        return f"""### {importance_marker} {memory.summary[:50]}
话题：{topics_str}
时间：{memory.created_at.strftime("%Y-%m-%d")}
详情：{memory.summary}
"""
```

    user_id: str,
    conversation_id: str,
    messages: list[BaseMessage],
) -> None:
    """记忆写入流程
    
    Args:
        user_id: 用户 ID
        conversation_id: 对话 ID
        messages: 对话消息列表
    """
    # === 1. 提取摘要和话题 ===
    summary_result = await memory_agent.extract_summary(messages)
    summary = summary_result.summary
    topics = extract_topics(summary)
    
    # === 2. 计算重要性评分 ===
    conversation_context = {
        "user_messages": [m.content for m in messages if m.type == "human"],
        "turn_count": len(messages) // 2,
    }
    importance_score = calculate_importance_score(summary, conversation_context)
    
    # === 3. 阈值判断 ===
    if importance_score < 0.3:
        logger.debug(f"Memory skipped: low importance ({importance_score})")
        update_cursor(user_id, conversation_id)
        return
    
    # === 4. 确定初始层级 ===
    initial_layer = (
        MemoryLayer.LTM if importance_score >= 0.7
        else MemoryLayer.STM
    )
    
    # === 5. 计算 Embedding ===
    embedding = embedding_adapter.embed(summary)
    
    # === 6. 写入数据库 ===
    session_summary = SessionSummary.create(
        id=str(uuid.uuid4()),
        conversation_id=conversation_id,
        user_id=user_id,
        summary=summary,
        embedding=embedding,
        topics=topics,
        importance_score=importance_score,
    )
    
    repo.create(session_summary)
    
    logger.info(f"Memory written: layer={initial_layer}, score={importance_score}, topics={topics}")
    
    # === 7. 更新游标 ===
    update_cursor(user_id, conversation_id)

## 十二、自适应策略（替代硬编码）

前文方案中存在多处硬编码，本节提供自适应替代方案。

### 12.1 硬编码问题汇总

| 硬编码项 | 现状 | 问题 |
|----------|------|------|
| 评分阈值 0.7/0.3 | 固定数值 | 不同用户/场景阈值可能不同 |
| 衰减速率 λ=0.3/0.05 | 固定参数 | 无法适应不同类型记忆 |
| 技术关键词库 | 预定义列表 | 新技术无法覆盖，维护成本高 |
| 评分维度权重 | 固定权重 | 不同场景权重应该动态调整 |

### 12.2 自适应替代方案

#### 方案 A：LLM 自决策（无硬编码阈值）

让 LLM 根据语义理解自行判断，而非硬编码评分公式：

```python
async def llm_judge_importance(summary: str, context: dict) -> dict:
    """LLM 自判断重要性
    
    不使用硬编码评分公式，让 LLM 根据语义理解判断。
    
    Returns:
        {
            "importance": "high/medium/low",
            "should_remember": True/False,
            "memory_layer": "long_term/short_term/none",
            "topics": ["rabbitmq", "kafka"],
            "reason": "判断理由",
        }
    """
    prompt = f"""你是一个记忆评估专家。请分析以下对话摘要，判断其重要性。

<对话摘要>
{summary}
</对话摘要>

<上下文>
- 对话轮次：{context.get("turn_count", 1)}
</上下文>

请判断：
1. 这段内容是否值得记住？（考虑：是否有具体结论、是否涉及偏好、是否是可复用经验）
2. 如果值得记住，应该放在长期记忆还是短期记忆？
3. 这段内容涉及哪些话题？
4. 给出重要性评级：high/medium/low

请以 JSON 格式回复。
"""
    
    response = await llm.invoke(prompt)
    return parse_json_response(response)
```

**优势**：
- 无固定阈值，LLM 理解语义后自行判断
- 可处理未见过的场景
- 无需维护关键词库

#### 方案 B：用户反馈驱动（动态调整权重）

借鉴 Mem0 的 Feedback Mechanism：

```python
class FeedbackDrivenMemory:
    """用户反馈驱动的自适应记忆"""
    
    # 反馈类型
    POSITIVE = "positive"
    NEGATIVE = "negative"
    
    def process_feedback(self, memory_id: str, feedback_type: str) -> None:
        """处理反馈
        
        反馈影响：
        - positive → importance +0.1, 可能升级到 LTM
        - negative → importance -0.2, 可能标记删除
        """
        memory = self.repo.get_by_id(memory_id)
        
        if feedback_type == self.POSITIVE:
            memory.importance_score = min(memory.importance_score + 0.1, 1.0)
            memory.feedback_score += 1
            
            # 多次正向反馈 → 升级
            if memory.feedback_score >= 3 and memory.memory_layer == STM:
                memory.memory_layer = LTM
                
        elif feedback_type == self.NEGATIVE:
            memory.importance_score = max(memory.importance_score - 0.2, 0.0)
            memory.feedback_score -= 1
            
            # 多次负向反馈 → 标记删除
            if memory.feedback_score <= -2:
                memory.marked_for_deletion = True
        
        self.repo.update(memory)
```

**优势**：
- 用户直接反馈，个性化调整
- 权重随反馈历史动态变化
- 重要记忆自然升级

#### 方案 C：统计驱动阈值（基于分布）

不使用固定阈值，基于用户历史数据分布：

```python
class DistributionBasedThreshold:
    """基于分布的自适应阈值"""
    
    def calculate_thresholds(self, user_id: str) -> dict:
        """计算用户个性化阈值
        
        原理：
        - 分析用户历史记忆的重要性分布
        - 使用分位数而非固定阈值
        
        Returns:
            {
                "ltm_threshold": 用户历史记忆 top 30% 的得分,
                "stm_threshold": 用户历史记忆 top 70% 的得分,
                "delete_threshold": 用户历史记忆 bottom 10% 的得分,
            }
        """
        memories = self.repo.get_all_memories(user_id)
        
        if len(memories) < 10:
            # 记忆太少，使用默认
            return {"ltm_threshold": 0.7, "stm_threshold": 0.3}
        
        scores = sorted([m.importance_score for m in memories])
        n = len(scores)
        
        return {
            "ltm_threshold": scores[int(n * 0.7)],   # top 30%
            "stm_threshold": scores[int(n * 0.3)],   # top 70%
        }
```

**优势**：
- 适应不同用户习惯
- 无固定数值，自动调整

#### 方案 D：LLM Criteria Retrieval（动态评分标准）

借鉴 Mem0 Criteria Retrieval，让 LLM 动态评分：

```python
class LLMCriteriaRetrieval:
    """LLM 动态评分召回"""
    
    async def score_memory_by_criteria(
        self,
        memory: SessionSummary,
        query: str,
        criteria: list[dict],
    ) -> float:
        """基于自定义标准评分
        
        示例 criteria（可动态调整）：
        [
            {"name": "relevance", "weight": 0.4},
            {"name": "importance", "weight": 0.3},
            {"name": "recency", "weight": 0.2},
        ]
        """
        prompt = f"""请根据标准对记忆评分。

<查询> {query} </查询>
<记忆> {memory.summary} </记忆>
<标准> {format_criteria(criteria)} </标准>

请对每个标准给出 0-10 评分，返回 JSON。
"""
        
        response = await llm.invoke(prompt)
        result = parse_json_response(response)
        
        # 计算加权平均
        scores = result.get("scores", {})
        total = sum(scores.get(c["name"], 0) / 10 * c["weight"] for c in criteria)
        
        return total
    
    def get_adaptive_criteria(self, query: str) -> list[dict]:
        """根据查询动态调整评分标准
        
        - 技术问题 → relevance 权重高
        - 偏好表达 → importance 权重高
        """
        if any(kw in query for kw in ["怎么", "如何", "解决"]):
            return [
                {"name": "relevance", "weight": 0.5},
                {"name": "importance", "weight": 0.3},
            ]
        elif any(kw in query for kw in ["我喜欢", "偏好"]):
            return [
                {"name": "relevance", "weight": 0.2},
                {"name": "importance", "weight": 0.5},
            ]
        return [
            {"name": "relevance", "weight": 0.4},
            {"name": "importance", "weight": 0.3},
        ]
```

**优势**：
- 标准可动态调整
- 场景敏感，精准召回

#### 方案 E：基于记忆强度的衰减

不使用固定衰减率，基于记忆强度：

```python
class StrengthBasedDecay:
    """基于记忆强度的自适应衰减"""
    
    BASE_DECAY_RATE = 0.1
    
    def calculate_decay_rate(self, memory: SessionSummary) -> float:
        """计算个性化衰减速率
        
        强记忆衰减慢，弱记忆衰减快
        
        公式：decay_rate = base_rate * exp(-strength)
        """
        # 记忆强度因子
        strength = (
            0.4 * min(memory.access_count / 10, 1.0) +
            0.4 * min(memory.feedback_score / 5, 1.0) +
            0.2 * time_strength(memory.created_at)
        )
        
        # 强记忆衰减慢
        return self.BASE_DECAY_RATE * math.exp(-2 * strength)
```

**优势**：
- 重要记忆衰减慢
- 自然遗忘不重要记忆

### 13.3 对比分析

#### PostgreSQL + pgvector 的优势

| 优势 | 说明 |
|------|------|
| **架构简单** | 一个数据库搞定所有，无需维护多个系统 |
| **事务支持** | 记忆写入可与其他操作在同一事务中 |
| **SQL 灵活性** | 支持复杂 JOIN、聚合、窗口函数 |
| **成熟稳定** | PostgreSQL 生态成熟，运维经验丰富 |
| **成本较低** | 无需额外向量数据库服务器 |
| **混合查询** | Payload 过滤 + 向量检索（pgvector 支持） |
| **关系数据整合** | 记忆与 conversation、user 表天然关联 |

#### PostgreSQL + pgvector 的劣势

| 劣势 | 说明 |
|------|------|
| **性能瓶颈** | 无 ANN 算法（HNSW/IVF），大规模检索慢 |
| **向量索引有限** | pgvector 只有 ivfflat/hnsw，性能不如专用库 |
| **向量维度限制** | pgvector 最大 2000 维（本项目 1024 在范围内） |
| **分布式能力弱** | 单机为主，难以横向扩展 |
| **向量压缩弱** | 无量化、压缩等高级功能 |
| **高并发场景** | 大量向量检索时 PostgreSQL 可能成为瓶颈 |

#### 专用向量数据库的优势

| 优势 | 说明 |
|------|------|
| **高性能检索** | ANN 算法（HNSW），毫秒级检索百万级向量 |
| **原生向量优化** | 索引、量化、压缩等高级功能 |
| **分布式部署** | 横向扩展，高可用 |
| **Payload 过滤原生** | filter + vector 混合检索性能优 |
| **向量维度支持广** | 可达 65536 维 |
| **实时更新** | 向量增删改性能优 |

#### 专用向量数据库的劣势

| 劣势 | 说明 |
|------|------|
| **架构复杂** | 需要维护两套系统（PostgreSQL + Vector DB） |
| **数据同步问题** | 记忆写入需要同步到两个系统 |
| **运维成本高** | 需要向量数据库专业知识 |
| **事务不完整** | 向量操作无法与关系数据在同一事务 |
| **关联查询弱** | 无法 JOIN 用户表、对话表 |
| **额外成本** | 需要额外的服务器资源 |

### 13.4 适用场景分析

| 场景 | 推荐方案 | 理由 |
|------|----------|------|
| **小规模（< 10 万记忆）** | PostgreSQL + pgvector | 性能足够，架构简单 |
| **中规模（10-100 万）** | PostgreSQL + pgvector + 优化 | ivfflat 索引可支撑 |
| **大规模（> 100 万）** | Qdrant/Milvus | ANN 算法性能优势明显 |
| **高并发检索** | Qdrant/Milvus | 专用向量库吞吐量高 |
| **复杂关联查询** | PostgreSQL + pgvector | SQL JOIN 优势 |
| **生产初期** | PostgreSQL + pgvector | 成本低，运维简单 |

### 13.5 当前项目的选择分析

**当前规模估算**：

| 用户规模 | 记忆数量估算 | PostgreSQL 能否支撑 |
|----------|--------------|---------------------|
| 100 用户 | ~5000 条 | ✓ 完全没问题 |
| 1000 用户 | ~50000 条 | ✓ 完全没问题 |
| 10000 用户 | ~500000 条 | ✓ ivfflat 索引可支撑 |
| 100000 用户 | ~500 万条 | ⚠️ 需要优化或考虑迁移 |

**当前项目状态**：
- 初期阶段，用户规模小
- PostgreSQL + pgvector 性能足够
- 架构简单，运维成本低

**建议**：
1. **短期**：继续使用 PostgreSQL + pgvector
2. **中期**：优化 pgvector 索引（创建 ivfflat/hnsw 索引）
3. **长期**：监控性能，若检索延迟 > 100ms，考虑迁移到 Qdrant

### 13.6 混合架构方案（可选）

如果未来需要迁移，可采用混合架构：

```python
class HybridMemoryStorage:
    """混合存储：PostgreSQL + Qdrant
    
    策略：
    - PostgreSQL：存储完整数据（元数据 + 文本 + 关系）
    - Qdrant：存储向量 + Payload（用于检索）
    - 双写同步，异步更新
    """
    
    def write_memory(self, memory: SessionSummary) -> None:
        """双写"""
        # 1. 写入 PostgreSQL（主存储）
        self.pg_repo.create(memory)
        
        # 2. 异步写入 Qdrant（检索优化）
        asyncio.create_task(self.qdrant_repo.upsert(
            point_id=memory.id,
            vector=memory.embedding,
            payload={
                "user_id": memory.user_id,
                "topics": memory.topics,
                "importance": memory.importance_score,
                "layer": memory.memory_layer,
            },
        ))
    
    def retrieve_memories(self, user_id: str, query_emb: list) -> list:
        """检索"""
        # 1. 从 Qdrant 检索（高性能）
        results = self.qdrant_repo.search(
            vector=query_emb,
            filter={"user_id": user_id},
            top_k=20,
        )
        
        # 2. 从 PostgreSQL 获取完整数据
        memory_ids = [r.id for r in results]
        memories = self.pg_repo.get_by_ids(memory_ids)
        
        return memories
```
