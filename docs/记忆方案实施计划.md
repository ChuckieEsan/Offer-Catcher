## 一、架构设计

### 1.1 Memory Agent 职责划分

```
┌─────────────────────────────────────────────────────────────┐
│ 主 Agent（Chat Agent）                                        │
│                                                              │
│ 职责：                                                        │
│ - 强加载 MEMORY.md 到 System Prompt                          │
│ - 可调用 load_memory_reference 加载详情                       │
│ - 可调用 search_session_history 检索历史                      │
│                                                              │
│ Tools：                                                       │
│ - load_memory_reference                                      │
│ - search_session_history                                     │
│ - load_skill                                                 │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ Memory Agent（独立 Agent）                                    │
│                                                              │
│ 职责：                                                        │
│ - 记忆检索：根据用户问题智能决定是否检索                         │
│ - 记忆更新：根据对话内容智能决定更新哪些记忆                     │
│                                                              │
│ Tools：                                                       │
│ - search_session_history（检索）                              │
│ - generate_session_summary（生成摘要）                        │
│ - save_session_record（保存到数据库）                         │
│ - update_preferences（更新偏好）                              │
│ - update_behaviors（更新行为）                                │
│ - update_memory_index（同步概要）                             │
│                                                              │
│ 决策能力：                                                     │
│ - 分析用户问题，判断是否需要检索历史                            │
│ - 分析对话摘要，判断是否有用户反馈                             │
│ - 智能决定更新顺序和跳过不必要的更新                            │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 流程对比

| 流程 | 原方案（Worker 硬编码） | 新方案（Agent 驱动） |
|------|------------------------|---------------------|
| **记忆注入** | 主 Agent 强加载 | 主 Agent 强加载（不变） |
| **记忆检索** | Worker 固定检索 | Memory Agent 智能决策是否检索 |
| **记忆更新** | Worker 固定 4 步 | Memory Agent 智能决策更新哪些 |

### 1.3 Memory Agent 决策示例

**记忆检索**：
```
用户问题："Python 的装饰器怎么用？"
Agent 决策：新问题，不需要检索历史 → 不调用 search_session_history

用户问题："上次讨论的 RAG 原理，能再讲一下吗？"
Agent 决策：涉及历史 → 调用 search_session_history(query="RAG 原理")
```

**记忆更新**：
```
对话摘要："用户询问了 Python 基础语法，无特殊反馈"
Agent 决策：无用户反馈 → 跳过 update_preferences，只更新 behaviors

对话摘要："用户反馈：解释不够详细，希望有代码示例"
Agent 决策：有反馈 → 调用 update_preferences，再更新 behaviors 和 MEMORY.md
```

## 五、Memory Agent 工具调用流程

### 5.1 记忆检索流程

```
Memory Agent 收到检索请求
    ↓
分析用户问题（query）
    ↓
决策：是否需要检索历史？
    ↓ 是
调用 search_session_history(query, top_k)
    ↓
返回检索结果
```

### 5.2 记忆更新流程

```
Memory Agent 收到更新请求
    ↓
1. generate_session_summary(messages, session_type)
    ↓ 返回 summary
2. save_session_record(user_id, conversation_id, summary, session_type)
    ↓
分析 summary 内容
    ↓
决策：是否有用户反馈？
    ↓ 是
3. update_preferences(user_id, feedback)
    ↓
4. update_behaviors(user_id, messages)
    ↓
5. update_memory_index(user_id)
    ↓
完成
```

## 七、关键设计决策

| 决策 | 原因 |
|------|------|
| **Memory Agent 独立** | 检索和更新逻辑复杂，需要智能决策 |
| **Tool 而非硬编码** | Agent 根据对话内容决定调用哪些工具 |
| **Prompt 外置** | 方便迭代优化决策逻辑 |
| **异步 Worker** | 不阻塞主流程，记忆是增强功能 |
| **Redis 锁** | 防止并发冲突 |

## 八、与原方案的差异

| 方面 | 原方案 | 新方案 |
|------|--------|--------|
| 检索触发 | 固定触发（消息 ≥ 10 字） | Agent 决策是否检索 |
| 更新流程 | 固定 4 步 LLM 调用 | Agent 决策调用哪些工具 |
| 可扩展性 | 添加步骤需改代码 | 添加 Tool + 更新 Prompt |
| Token 消耗 | 每次都调用 4 次 | 智能跳过不必要调用 |
