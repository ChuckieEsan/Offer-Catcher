# 记忆模块实施计划

> 基于《记忆模块最终设计方案.md》制定  
> 版本：v1.0  
> 更新时间：2026-04-13

---

## 一、项目概述

### 1.1 目标

为 Offer-Catcher 实现长期记忆模块，支持：
- 用户偏好和行为模式的持久化
- 会话历史的语义检索
- 用户自定义 Skill 管理

### 1.2 核心设计决策

| 决策 | 说明 |
|------|------|
| 记忆 = Skill | 遵循 Agent Skills 标准 |
| MEMORY.md + references | 渐进式披露 |
| session_summaries 独立表 | 数据 vs 规则分离 |
| Worker 直接更新 Checkpointer | 无需中间缓存 |

### 1.3 总工期

| Phase | 内容 | 工期 |
|-------|------|------|
| Phase 1 | 核心记忆结构 | 3.5 天 |
| Phase 2 | 写入流程 | 4.5 天 |
| Phase 3 | 用户自定义 Skill | 3.5 天 |
| **总计** | | **11.5 天** |

## 八、风险与缓解

### 8.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| Checkpointer API 不明确 | Worker 更新失败 | 提前查阅 LangGraph 文档，验证 API |
| pgvector 索引性能差 | 检索延迟 | 使用 ivfflat 索引，测试验证 |
| LLM Prompt 效果不佳 | 摘要质量差 | Prompt 迭代优化，A/B 测试 |

### 8.2 业务风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 用户不使用记忆功能 | ROI 低 | 前端入口明显，引导用户 |
| 记忆内容过长 | Token 成本高 | 容量控制，LLM 精简 |
