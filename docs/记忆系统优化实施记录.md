## 优化背景

### 问题分析

| 问题 | 影响 | 优先级 |
|------|------|--------|
| 固定 10 条消息窗口 | 可能丢失重要上下文或超出 Token 限制 | 高 |
| 无长期记忆存储 | 用户偏好跨会话丢失 | 高 |
| 无 Token 预算管理 | 可能触发 LLM 上下文限制 | 中 |

### LangChain 最佳实践参考

根据 LangChain 官方文档 (2025):

1. **短期记忆**: 使用 `@before_model` middleware 动态裁剪消息
2. **长期记忆**: 使用 `PostgresStore` 存储用户偏好和会话摘要
3. **上下文注入**: 使用 `@dynamic_prompt` 动态生成 System Prompt

## 实施内容

### 1. 短期记忆优化 - 消息裁剪

**文件**: `backend/app/agents/graph/nodes.py`

**实施内容**:
```python
def _apply_message_trimming(
    messages: list[BaseMessage],
    max_messages: int = 10,
    max_tokens: int = 8000
) -> list[BaseMessage]:
    """应用消息裁剪策略

    策略：
    1. 保留 SystemMessage（如果有）
    2. 保留最近 N 条消息
    3. 如果 token 超限，进一步裁剪
    """
```

**关键逻辑**:
- 分离 system message 和普通消息
- 优先保留最近的对话
- Token 超限时自动裁剪到一半

### 3. 在工具中访问长期记忆

```python
from langchain.tools import ToolRuntime, tool

@tool
def get_user_preferences(runtime: ToolRuntime) -> str:
    """获取用户偏好设置"""
    user_id = runtime.context.get("user_id")
    if not user_id or not runtime.store:
        return "用户未登录"

    prefs = runtime.store.get(
        ("users", user_id, "preferences"),
        "user_preferences"
    )

    if prefs and prefs.value:
        return (
            f"语言：{'中文' if prefs.value['language'] == 'zh' else 'English'}\n"
            f"难度：{prefs.value['difficulty']}\n"
            f"单次练习：{prefs.value['practice_batch_size']} 道题"
        )
    return "暂无偏好设置"
```

### 4. 验证脚本

