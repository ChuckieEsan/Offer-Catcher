# 后续优化

## 优化2: 入库前检查 MQ 状态

### 问题背景

当前入库流程中，如果 RabbitMQ 队列里有一个待处理的任务（有 question_id 但尚未生成答案），再次录入相同题目时：
- Qdrant 中该 question_id 的记录存在但无答案
- 会再次发送 MQ 任务，导致重复

### 解决方案

#### 方案 A：本地内存追踪（简单）

使用 Python 内存 dict 追踪已发送的 question_id：

```python
# 在 producer.py 中
_sent_task_ids: dict[str, datetime] = {}

def publish_task(self, task: MQTaskMessage, retry: int = 3) -> bool:
    # 检查是否已发送过
    if task.question_id in _sent_task_ids:
        logger.info(f"Task {task.question_id} already sent, skipping")
        return False

    # 发送任务...
    _sent_task_ids[task.question_id] = datetime.now()
```

**缺点**：服务重启后会丢失状态

#### 方案 B：Redis 状态追踪（推荐）

1. 发送 MQ 任务时，写入 Redis
2. Worker 完成时，删除或标记为完成
3. 入库前检查 Redis 状态

```python
# 发送任务时
redis.setex(f"mq:task:{question_id}", 3600, "pending")

# 入库前检查
if redis.exists(f"mq:task:{question_id}"):
    # MQ 中有待处理任务，跳过发送
    pass
```

#### 方案 C：使用 RabbitMQ 的消息去重（需付费插件）

使用 RabbitMQ 的消息去重插件或消费者端的幂等性处理。

---

### 实现建议

推荐使用 **方案 B（Redis）**，需要：
1. 添加 Redis 依赖到 pyproject.toml
2. 在 producer.py 中添加 Redis 写入逻辑
3. 在入库逻辑中检查 Redis 状态
